🔥 You caught the tricky bit. Tailwind is basically a **utility-first layer** that compiles into CSS. By default:

*   **Tailwind classes win** if they’re applied directly on the element.
    
*   **Custom CSS wins** if you either:
    
    1.  Extend Tailwind with **plugins** (adds utilities you can call like `font-heading-lowercase`), or
        
    2.  Write a **layer override** in your CSS file (`@layer components` or `@layer utilities`).
        

So yeah — if you just slap a CSS rule in `globals.css` without thinking, Tailwind’s compiled classes can stomp it. But you _can_ make it play nice.

## 💡 For Your H1 "lowercase + bold-first-word" Quirk

You want:

*   All H1s = lowercase.
    
*   If 1–2 words → first bold, rest normal.
    
*   Else = all bold.
    

Tailwind doesn’t know context (word count), so we do **CSS with pseudo-selectors**:

CSS

```
@layer components {  h1 {    @apply font-heading lowercase font-bold;  }  h1.two-words span:first-child {    @apply font-bold;  }  h1.two-words span:last-child {    @apply font-normal;  }}
```

Then in your HTML/JSX:

HTML

```
<!-- Single word, auto bold --><h1 class="text-4xl">swot</h1><!-- Two words, first bold, second normal --><h1 class="two-words text-4xl">  <span>swot</span> <span>analysis</span></h1><!-- Longer phrase, all bold --><h1 class="text-4xl">operational health audit report</h1>
```

* * *

## ⚡ Why This Works

*   `@layer components` ensures your custom rules get compiled **inside Tailwind’s cascade** — not stomped later.
    
*   You still use Tailwind utilities (`font-bold`, `lowercase`, etc.), so it’s consistent.
    
*   The `two-words` class is your “escape hatch” for that funky bold-first-word rule.
