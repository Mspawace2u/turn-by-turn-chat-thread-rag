This framework was developed to address the complex operational reality at NeuroGym — a cross-functional, neurodivergent team spread across multiple time zones, with varying attention spans, executive function patterns, and processing styles.

The goal: replace rigid “productivity porn” systems with a **blended agile framework** that gives enough structure to deliver _on time_, without burning people out or forcing neurotypical expectations on neurodivergent contributors.

This framework lives between classic Scrum and async-first project flow, optimized for:

*   🧠 Brain-based rhythm matching
    
*   🌍 Global async collaboration
    
*   🔁 Flexible sprint cycles with status tracking
    
*   🔒 Reduced overwhelm, increased ownership
    

* * *

## 🧠 Core Principles

| Principle | Description |
| --- | --- |
| 🧭 Brain Before System | Framework adapts to how people _naturally work_, not vice versa |
| 🔄 Iterative Cadence | Shorter sprints (5–7 working days) with async midpoints instead of forced stand-ups |
| 🕹️ Choice-Based Task Ownership | People pick tasks that match their energy band, skill zone, and attention budget |
| 🎛️ Multiple Visibility Modes | Views built for “executive brain,” “detailed implementer,” and “big-picture strategist” |
| 🧩 Embedded SOPs & Context | No more hunting for directions — tasks include the _how_ and _why_ in one place |

* * *

## 🌍 Time Zone-Aware Team Structure

| Region | Roles | Scheduling Logic |
| --- | --- | --- |
| 🇺🇸 PST–CST | Execs, Creatives, PMs | Morning sync possible; async by afternoon |
| 🇲🇽 Mexico / Central America | Tech, Copy, CX | Works well with PST overlap |
| 🇪🇺 Europe | Design, Strategy Support | Async-first; end-of-day review windows |
| 🇮🇳 India/APAC | Optional contractors | Used only for batch tasks, not sprint-based work |

> 💡 Async workflows allow for “follow-the-sun” progress without relying on real-time standups or bottlenecks.

* * *

## 🔧 Sprint Cadence

| Phase | Duration | Description |
| --- | --- | --- |
| ⏱️ Planning | 1 day (async or live) | Owner sets goals, team selects tasks via “energy band” matching |
| 🚧 Execution | 5–7 working days | Daily check-ins via board updates — no forced meetings |
| 🧠 Mid-Sprint Pulse | 15-min async Loom/Slack share | Quick “where I’m stuck / where I’m flowing” updates |
| 📈 End-of-Sprint Review | 30-min async deck or dashboard walk-through | Measurable outcomes, blockers, and wins shared to team lead |
| 🔁 Rollover Cycle | 1 day | Tasks auto-roll into next sprint w/ notes unless manually reassigned |

* * *

## 🧩 Task Flow Anatomy

Each task includes:

*   🧠 What is the desired outcome?
    
*   🔍 Why does this matter right now?
    
*   📋 How exactly do I complete it?
    
*   📆 When is it due (with buffer)?
    
*   🎨 What “brain style” does it best suit? (Focus, creative, admin, reactive)
    

> Tasks are tagged by **Vibe Code** or **Attention Band** to match the team’s real-time energy distribution.

* * *

## 📊 Visibility & Roles

| Role | View Type | Responsibilities |
| --- | --- | --- |
