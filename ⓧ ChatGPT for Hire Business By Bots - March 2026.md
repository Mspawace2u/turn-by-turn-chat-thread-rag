### **Function 3: executeScrape(queries)**

**Purpose:** Run Google Dork queries via Puppeteer, extract results.

**Logic:**

```
1. Launch Puppeteer browser (headless: true)
2. FOR each query in queries:
   a. Navigate to Google search with query
   b. Wait for results to load (waitForSelector)
   c. Extract from each result:
      - Title
      - URL
      - Snippet text
   d. Add 5-10 second random delay between queries (anti-detection)
   e. IF Google CAPTCHA detected:
      - Log warning
      - Pause 60 seconds
      - Retry once
      - IF still blocked: Skip query, continue to next
3. FOR each result URL:
   a. Navigate to the actual post/comment page
   b. Extract:
      - Author name/handle
      - Author profile URL
      - Full comment/post text
      - Company name (if visible)
      - Job title (if visible)
   c. Add 3-5 second random delay between page visits
4. Close browser
5. RETURN array of raw lead objects
```

**Rate Limiting:**

```javascript
const SCRAPE_CONFIG = {
  delayBetweenQueries: { min: 5000, max: 10000 },  // ms
  delayBetweenPages: { min: 3000, max: 5000 },      // ms
  maxQueriesPerRun: 20,
  maxPagesPerRun: 50,
  captchaRetryDelay: 60000,                          // ms
  userAgentRotation: true
};
```

**Error Handling:**

- Navigation timeout: Skip URL, log warning, continue  
- CAPTCHA: Pause 60s, retry once, then skip  
- Rate limited: Stop scraping, process what we have  
- Browser crash: Restart browser, resume from last query

### **Function 4: scoreLeads(rawLeads)**

**Purpose:** Use OpenAI to score pain intensity and categorize each lead.

**Logic:**

```
1. FOR each raw lead:
   a. Send to OpenAI GPT-4o-mini with scoring prompt:
   
      PROMPT:
      "Analyze this social media comment for buying intent signals.
      
      Comment: {commentText}
      Platform: {platform}
      Author: {name}
      
      Score this person 1-10 on likelihood they would PAY for 
      AI/ChatGPT consulting help. Consider:
      - Urgency of their pain (are they desperate or just curious?)
      - Budget signals (do they mention business, revenue, clients?)
      - Decision-maker signals (are they the boss or an employee?)
      - Specificity of need (vague complaint vs specific problem?)
      
      Also categorize their primary pain into ONE of these:
      - AI Overwhelm
      - Manual Work
      - No Strategy
      - Budget Concerns
      - Time Constraints
      - Tech Confusion
      
      Also extract (if detectable):
      - Company name
      - Job title/role
      
      Respond in JSON format:
      {
        'painScore': number,
        'painCategory': string,
        'company': string or null,
        'roleTitle': string or null,
        'reasoning': string (one sentence why this score)
      }"
      
   b. Parse JSON response
   c. IF painScore >= MIN_PAIN_SCORE_TO_SAVE (default 5):
      - Add to qualified leads array
   d. IF painScore < MIN_PAIN_SCORE_TO_SAVE:
      - Discard (don't waste Notion API calls)
   e. Add 500ms delay between API calls (rate limiting)

2. RETURN array of scored, qualified leads
```

**API Cost Control:**

```javascript
const SCORING_CONFIG = {
  model: 'gpt-4o-mini',           // Cheapest capable model
  maxTokensPerCall: 200,           // Keep responses short
  delayBetweenCalls: 500,          // ms
  maxCallsPerRun: 50,              // Hard cap
  dailyBudgetCap: 1.00             // USD - stop if exceeded
};
```

**Error Handling:**

- API rate limit: Wait 30s, retry  
- Invalid JSON response: Retry once with stricter prompt  
- API key invalid: MacOS popup alert, exit  
- Budget cap hit: Stop scoring, save what we have

### **Function 5: checkForDuplicates(scoredLeads)**

**Purpose:** Don't write duplicate leads to Notion.

**Logic:**

```
1. FOR each scored lead:
   a. Query Notion database: 
      Filter where profileUrl equals lead.profileUrl
   b. IF match found:
      - Skip this lead
      - Log: "Duplicate skipped: {name}"
   c. IF no match:
      - Keep in array
2. RETURN deduplicated leads array
```

### **Function 6: writeToNotion(qualifiedLeads)**

**Purpose:** Write all qualified, deduplicated leads to Notion database.

**Logic:**

```
1. FOR each qualified lead:
   a. Create new page in "GrowthOps Leads Pipeline" database
   b. Map fields:
      - name: lead.name (title)
      - profileUrl: lead.profileUrl
      - platform: lead.platform
      - commentText: lead.commentText
      - painScore: lead.painScore
      - painCategory: lead.painCategory
      - status: "New"
      - dateFound: new Date().toISOString()
      - sourcePostUrl: lead.sourcePostUrl
      - company: lead.company || ""
      - roleTitle: lead.roleTitle || ""
      - agentLastUpdated: new Date().toISOString()
   c. Log: "Lead added: {name} (Score: {painScore})"
   d. Add 300ms delay between writes (Notion rate limit: 3 req/sec)
2. RETURN count of leads written
```

**Error Handling:**

- Notion rate limit (429): Wait 1s, retry  
- Write failure: Log error, skip lead, continue  
- Database not found: Re-run initializeDatabase()
