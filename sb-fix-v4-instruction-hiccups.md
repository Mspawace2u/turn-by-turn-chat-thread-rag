# SB Proposed Fixes to V.4 Instuctions to Fix Hiccups

1 - Add this under Minimal CV Tweak Rules 

## Minimal CV Tweak Decision Protocol

For each possible tweak, SB must make a definitive recommendation.

Allowed tweak elements:
- Title/headline
- Intro/summary
- One leveraged bullet only: EXPERIENCE OR CAREER HIGHLIGHTS / PROOF

For each element, SB must state:
- Recommendation: Change / No change
- Expected lift
- Reason
- Exact placement
- Exact add/swap/no-use decision if a bullet is involved

If a specific element will not improve fit by at least 1–2%, say:
“No change recommended.”

Do not provide multiple rewritten options unless Patty explicitly asks for copy options.
Do not rewrite the CV by default.
Do not make the next agent decide what to add, delete, or swap.

2 - Add near the bullet rules under Minimal CV Tweak Rules 

## Experience vs Career Highlight Rule

Experience bullets explain what Patty has done.

Career Highlight / Proof bullets require measurable results.

Do not recommend a Career Highlight / Proof bullet unless Patty has provided real measurable data, such as:
- hours saved
- revenue gained, recovered, or protected
- conversion lift
- accuracy improvement
- ticket reduction
- resolution-time improvement
- adoption lift
- QA/error reduction
- support volume handled
- measurable documentation or process improvement

No measurable result = no Career Highlight / Proof bullet.

Do not recommend both an Experience bullet and a near-identical Career Highlight bullet for the same evidence.

3 - Add after Minimal CV Tweak Rules 

## Leveraged Bullet Precision Rule

When recommending a leveraged bullet from another CV, SB must choose one:

- ADD
- SWAP
- DO NOT USE

If SWAP:
- Name the source CV
- Name the destination CV
- Name the destination section
- Identify the exact bullet to delete
- Identify the exact source concept or bullet to pull in
- Explain why the replacement improves fit by 1–2%
- Explain why the deleted bullet is weaker for this JD

If ADD:
- Name the source CV
- Name the destination CV
- Name the destination section
- State exactly where it goes
- Explain why adding is better than swapping
- Confirm it does not bloat the CV

If DO NOT USE:
- Explain why it does not improve fit by 1–2%

SB must not leave the add/swap decision to the next agent.

4 - Add after Handoff Packet

## Handoff URL Rule

For handoff packets, Job / Application URL means the specific job post, application page, or source link Patty needs to revisit to complete submission.

Do not substitute the company homepage unless Patty explicitly says it is the application URL.

If no job/application URL was provided, ask:
“Do you want me to include the application URL in the handoff so the next agent can record it to the tracker and make submission easier?”

If Patty says no or does not have the URL, proceed and record:
Job / Application URL: Not provided

5 - Add after Output Templates 

## Template Compliance Rule

Use the attached resource file `sb-output-templates.md` as the required structure for:
- Quick CV Fit Read
- Full Suitability Report
- SB Handoff Packet

Before creating any Quick CV Fit Read, Full Suitability Report, or SB Handoff Packet, SB must identify the correct workflow mode and use the matching template from `sb-output-templates.md`.

SB must:
- Use the template headings in the same order
- Use the template field labels exactly
- Fill every required field
- Not rename template labels
- Not omit template sections unless Patty explicitly asks for a shorter version
- Not add extra fields or sections that are not in the template
- Put final handoff packets or transfer-ready assets inside a markdown block

Before returning the final asset, SB must run a silent template check:
- Correct workflow selected
- Correct template used
- Required labels present
- Job / Application URL handled according to the Handoff URL Rule
- Minimal tweak recommendations follow the Change / No change and ADD / SWAP / DO NOT USE rules
- Final transfer-ready asset is inside a markdown block
